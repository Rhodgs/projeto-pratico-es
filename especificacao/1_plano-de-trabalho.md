## 1. Introdução e Contexto 

Este projeto consiste no planejamento e especificação do sistema Jornada Verde, originalmente concebido na disciplina de Interação Humano-Computador (IHC). O objetivo desta etapa é transformar as ideias iniciais e o protótipo em artefatos concretos de planejamento de desenvolvimento

O sistema busca resolver o problema de falta de compartilhamento de informação sobre o assunto de conscientização ambiental

---

## 2. Objetivos do Projeto 

  Aplicar conceitos fundamentais da engenharia de software no planejamento da solução.

  Realizar a especificação de requisitos e a divisão de tarefas entre os membros da equipe.

  Estabelecer um fluxo de acompanhamento ágil para garantir a evolução do produto.

## 4. Metodologia de Trabalho 

O grupo utilizará práticas inspiradas no framework Scrum para a gestão do desenvolvimento:

  Gestão de Tarefas: Utilização do GitHub Projects para organizar o Backlog do Produto e o Backlog do Sprint.

  Acompanhamento Diário (Daily Scrum): Registro de atividades no Notion, respondendo o que foi feito, o que será feito e impedimentos.

  Frequência: As atualizações ocorrerão de 3 a 4 vezes por semana.
